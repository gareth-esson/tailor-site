Annotations
A6 Simple Mode toggle: Controls both the definition (Section 3) and the full explainer (Section 4). Same component as C1. Should remain accessible as reader scrolls. Persists via localStorage.
A7 Glossary tooltips: The full explainer (Section 4) may contain inline links to other glossary terms, tooltipped the same way as on question pages. First occurrence only.
A8 Signposting: Conditional — only appears if the term's topic warrants support service links. Same component as C1 but not guaranteed on every page.
A15 Related content cards: Two variants used here — glossary variant (Section 6) for related terms, and question variant (Section 7) for referencing questions. Same card component as C1, different content types.
A16 Topic tags: Links to the landing page(s) this term belongs to. Same component as C1.

Design decisions flagged

Definition prominence: The definition (Section 3) is the tooltip text shown standalone. It should be visually distinct from the explainer — a pull-out block, larger type, or a coloured background. This is what the visitor came for; the explainer is the depth.

Simple Mode scope: The toggle controls both the definition and the explainer. When toggled, both swap simultaneously. Related term cards (Section 6) also swap their definitions to match the active mode.

Referencing questions: Auto-populated — every question whose answer contains a tooltip for this term. This could be a long list for common terms (e.g. "consent" might appear in 20+ questions). Consider a "Show more" limit — 3–5 visible, rest behind a toggle.

CTA destination: Links to the C3 landing page for this term's primary topic. If the term spans multiple topics, use the primary one. The topic tags (Section 10) cover the full set.
