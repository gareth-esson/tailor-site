Annotations
→ Three filter dimensions: Topic (the 22 landing page topics), category (practical teaching / curriculum & policy / bridging content), and audience (teachers / school leaders / parents). These can be combined — e.g. "show me practical teaching posts about consent for teachers." Filters are additive.
→ Category chips: The quick-filter chips below the dropdowns offer the blog content categories as a fast toggle. "All" is the default. These are an alternative interaction to the dropdown — tapping a chip updates the list the same way the dropdown does.
→ Tile layout: 2-column grid on desktop, single column on mobile. Tiles give each post equal visual weight and scale well as the library grows. The consistent height means tags always sit at the bottom — no ragged edges.
→ Excerpts: Kept short (1–2 lines) to fit the tile format. These should be hand-written opening lines, not auto-generated summaries. Gareth's blog copy should front-load the hook.
→ Author omitted: Since every post is by Gareth, the author name is dropped from the tile to save space. It's shown on the C4 post page itself.
→ Topic tags + audience label: Each tile shows its topic tags (bottom) and audience label (meta line). Both serve double duty — scanability for the reader, and clickable links to C3 landing pages for the tags.

Design decisions flagged

Filters: dropdowns vs chips? The wireframe shows both — dropdowns for topic and audience (which have many options), chips for category (which only has 3 options). Could simplify to chips-only if the filter set is small enough, or dropdowns-only for consistency. With 22 topics, a dropdown is more practical than 22 chips.

Tile sizing: All tiles should be equal height within each row. The title and excerpt vary in length, so the excerpt area is the flex element — tags pin to the bottom. If a title is long enough to wrap to 3 lines, the excerpt can shorten to 1 line to compensate.

Empty filter states: If a filter combination returns no results, show: "No posts matching these filters. Try broadening your search, or browse all posts." Reset filters link.

Pagination vs load-more: Pagination shown. At ~20 posts, the index may not need pagination at launch (~10 tiles per page = 2 pages). But the template should support it. With tiles, 8–10 per page feels right (4–5 rows of 2).

Featured/pinned post: Not in the brief, but a pinned tile could span both columns at the top — a "latest" or "featured" post that's always visible regardless of filters. Useful for timely content like the DfE guidance changes post.

Mobile: Tiles stack to a single column, full width. The grid collapses naturally.
