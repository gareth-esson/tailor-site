Annotations
→ Grouped, not flat: Results are grouped by content type rather than shown as a single flat list. This helps users find the kind of content they want — a teacher looking for a topic hub won't have to scroll past 7 question results to find it.
→ 5-result cap per group: Each group shows a maximum of 5 results initially. "Show all" expands the group inline. This keeps the page scannable when a query matches many items across multiple types.
→ Excerpts: Each result shows a snippet of the matching content. Where possible, highlight the matching term in the excerpt (bold or background colour). This helps the user judge relevance without clicking through.
→ Topic tags on questions: Question results show their topic tag so the user can see context at a glance — "What does consent mean?" tagged "Relationships" vs tagged "Sex and the Law" tells you the angle.
→ No results escape hatch: The empty state links to the Okay to Ask landing (B8) or topic hubs as a fallback. Never a dead end — always a next step.
→ URL structure: /search?q={query} — bookmarkable, shareable. The query is in the URL so back/forward navigation works naturally.

Design decisions flagged

Group order: Same as the dropdown: Questions → Glossary → Topics → Blog → Services. This could be made dynamic (show the group with the most results first), but consistent ordering is simpler for users to learn.

Search term highlighting: Matching terms in titles and excerpts should be visually highlighted (bold, underline, or background colour). This is standard in search UIs and helps users scan quickly.

"Okay to Ask" label on questions: The brief says question results should carry an "Okay to Ask" label. This differentiates them from other content types for users who arrived via the teacher-facing side of the site and may not be familiar with the sub-brand.

Refine in place: The search bar at the top of B10 is fully functional — editing the query and pressing Enter re-searches without leaving the page. The A5 dropdown can also appear here for quick refinement.

Mobile: Everything stacks. Groups are full-width. The search bar remains at the top, prominent and editable. No horizontal layout changes needed.
