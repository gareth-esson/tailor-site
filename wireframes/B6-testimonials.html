Annotations
→ Two testimonial types: Pull quotes are short and scannable — ideal for quick social proof. Case-study blocks are longer and narrative — they tell a story about the impact of the work. Both types are needed; the mix keeps the page from feeling like a wall of identical quotes.
→ Service tagging: Every testimonial is tagged by service line (delivery, drop days, training, consultancy). This enables filtering so visitors arriving from a specific service page can see relevant proof immediately. Tags also help Gareth organise testimonials as the collection grows.
→ Attribution matters: School name + person's role is the gold standard. "PSHE Lead, Oakfield Academy" is more credible than an anonymous quote. Where full attribution isn't possible, role + key stage still adds credibility.
→ Filter behaviour: "All" shows everything. Selecting a service chip filters to just that service's testimonials. Only one filter active at a time. If a service has no testimonials yet, the chip can be hidden or disabled rather than showing an empty state.
→ Growth path: This page starts small (maybe 4–6 testimonials at launch) and grows over time. The structure supports scaling — the filter becomes more useful as the collection builds. Case studies can be added as relationships with schools deepen.

Design decisions flagged

Testimonial density: Pull quotes could appear in a single column (full-width, stacked) or a masonry/2-column grid. Single column is simpler and more readable; grid is more visual and lets you show more on screen. Consider starting single-column and moving to grid as the collection grows.

Case studies: Could be expandable (show first 2 lines, click to expand) or always fully visible. If there are only 1–2 case studies at launch, show them in full. If the page grows to 5+, consider expandable.

Photo/logo: If schools provide logos or photos, these could appear alongside testimonials for visual credibility. Not essential at launch but worth planning for in the component design.

Deep-link from service pages: Service pages (C5) and the services hub (B5) can link to this page with a service filter pre-selected via URL parameter (e.g., `/testimonials?service=training`). This creates a seamless "see what others say" flow from any service page.

Mobile: Filters scroll horizontally as a chip row. Testimonials stack full-width. Case-study blocks stay full-width.
